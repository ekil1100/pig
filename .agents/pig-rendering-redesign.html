一眼看懂

核心判断

问题不是单个 ANSI 序列，而是 Pig 同时存在 transcript append、streaming stdout、footer overlay 和 scripted full-frame 多套渲染路径。

目标模型

改成 Pi-style scrollback-preserving document renderer：所有 UI 都先变成 logical lines，再由唯一 renderer 写 terminal。

优先修复

先解决 terminal ownership 和 frame 更新，再做组件化和更漂亮的 Markdown/footer。

最大收益

streaming 内容不再重复，busy status 和输入行不会插进 assistant markdown，resize/wrap 由统一 renderer 处理。

重点：任何 provider、tool、agent event 都不能直接写 ANSI。它们只能更新状态；terminal 写入只能由 renderer 统一执行。

根因诊断

表象当前冲突路径重设计后的归属
assistant streaming 时旧快照留在 scrollback，内容重复streaming delta 直接写 stdout，同时 transcript 也 appenddelta 只更新 AssistantMessage component；renderer 从完整 logical lines diff
... running 和 > 输入行插入 markdown 中间footer/prompt 用 save/restore cursor 伪固定到底部footer/editor/status 都是 document 末尾组件
复杂 markdown、表格、目录树 streaming 中横向偏移writer 不统一，wrap/width 不在单一 layout 层收敛所有 component 输出必须满足 visible width 不超过 terminal width

渲染循环和 cursor 模型

事件循环

while running:
  drain input events
  drain agent event queue
  apply events to InteractiveState
  if state changed:
    requestRender()
  renderer.renderIfDue(stateToFrame(state))

render 请求只置脏，streaming 忙时按 16-33ms coalesce；输入编辑可以立即渲染。

cursor 定位

  1. Editor 在 logical cursor 位置插入 zero-width marker。
  2. Renderer 查找 marker，计算 display width。
  3. 剥掉 marker，渲染 frame。
  4. 最后把硬件光标定位到 marker 对应位置。
这替代 \x1b7 / \x1b8 save/restore cursor 修补，避免 prompt 被 streaming markdown 撞散。

迁移路线

停止扩展 LiveRenderer

标记待删除，停止向 streamTextItem / renderStreamingFooter 添加修补逻辑。验收标准是后续 PR 不再增加 save/restore cursor footer 补丁。

引入 logical document renderer

新增 tui/renderer.zig，维护完整 logical lines、viewport state 和 single-buffer flush。覆盖 append、middle-line change、shrink、resize、cursor marker。

统一 interactive render path

InteractiveApp.renderFrame() 生成完整 logical document；live 用 TerminalRenderer，scripted 用同一 frame snapshot。删除旧的 streaming/live 高度状态。

组件化 transcript/status/editor/footer

把 transcript 拼接抽到 component/view 层，Editor 输出 cursor marker，Tool/status/busy/footer 都作为普通 component。

真实 viewport regression

新增 pure Zig virtual terminal 或 opt-in headless xterm runner，固化长 markdown streaming、tool lines、running status、prompt/footer 场景。

不要做什么

不要直写 terminal

provider parser 或 AgentRuntime 不直接输出 terminal 文本。

不要 delta writeAll

streaming 分支不再直接 writeAll(delta)。

不要 save/restore prompt

主输入行不再靠 \x1b7 / \x1b8 维护。

不要双 renderer

不要同时维护 scrollback append renderer 和 fixed-bottom renderer。

落地顺序：先做 tui/renderer.zig 的 Pi-style logical-line diff，再切 live interactive，最后组件化和美化。
